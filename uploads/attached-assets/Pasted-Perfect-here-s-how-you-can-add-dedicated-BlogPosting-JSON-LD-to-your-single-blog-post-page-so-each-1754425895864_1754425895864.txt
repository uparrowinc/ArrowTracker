Perfect — here’s how you can add dedicated BlogPosting JSON-LD to your single blog post page so each article gets its own rich snippet in Google.

Example for BlogPost.tsx (or whatever your single-post component is called)
Place this inside your return (near the bottom but before </div>), using the same dangerouslySetInnerHTML pattern:

{/* JSON-LD for Single Blog Post SEO */}
<script
  type="application/ld+json"
  dangerouslySetInnerHTML={{
    __html: JSON.stringify({
      "@context": "https://schema.org",
      "@type": "BlogPosting",
      "headline": post.title,
      "description": post.excerpt || maskContent(post.content, 160),
      "url": `https://uparrowinc.com/blog/${post.slug}`,
      "mainEntityOfPage": {
        "@type": "WebPage",
        "@id": `https://uparrowinc.com/blog/${post.slug}`
      },
      "datePublished": post.publishedAt,
      "dateModified": post.updatedAt || post.publishedAt, // use actual last edit if available
      "image": post.featuredImage
        ? [post.featuredImage]
        : ["https://uparrowinc.com/default-blog-image.jpg"],
      "author": {
        "@type": "Person",
        "name": post.author || "Up Arrow Inc"
      },
      "publisher": {
        "@type": "Organization",
        "name": "Up Arrow Inc",
        "logo": {
          "@type": "ImageObject",
          "url": "https://uparrowinc.com/logo.png"
        }
      },
      "keywords": post.tags
        ? post.tags.split(",").map(tag => tag.trim())
        : [],
      "articleSection": selectedCategory || "General",
      "wordcount": post.content
        ? post.content.replace(/<[^>]*>/g, "").split(/\s+/).length
        : 0
    })
  }}
/>
Key SEO Benefits
Rich Snippets for each article — Google can display:

Headline

Image

Date Published

Publisher Logo

Keywords

mainEntityOfPage — explicitly ties the article to its permalink.

wordcount & articleSection — extra ranking signals for Google News & Discover.

dateModified — encourages Google to re-crawl when you update content.
