How to Fix This in Your Replit Code
Assuming your RSS generation is in server/rss.ts (from the debug logs), the issue is that the enclosure tag uses proper sanitization (replacing spaces with _, removing apostrophes, etc.), but the content:encoded link parser is doing something simpler (just stripping spaces without adding _).

Locate the Cleaning Logic:

Search for functions like sanitizeFilename, cleanFilename, or regex replaces involving /\s+/g or /[^\w\-_.]/g.
There are probably two spots:

One for <enclosure> (correct: adds _ for spaces).
One for parsing [audio:...] tags in blog content for <content:encoded> (buggy: removes spaces but doesn't add _).




Unify the Cleaning Function:


Create or update a single reusable function for filename sanitization. Here's a solid Node.js example you can copy-paste into server/rss.ts (adjust as needed):
typescriptfunction sanitizeFilename(filename: string): string {
  // Remove query params if present (e.g., &title=...)
  filename = filename.split('&')[0].trim();
  
  // Replace spaces with underscores
  filename = filename.replace(/\s+/g, '_');
  
  // Remove apostrophes and other invalid chars, replace with underscore if needed
  filename = filename.replace(/'/g, '').replace(/[^\w\-_.]/g, '_');
  
  // Ensure it ends with .mp3 if missing
  if (!filename.endsWith('.mp3')) {
    filename += '.mp3';
  }
  
  return filename;
}
